Favicon
- Defined in <head> section of page (must be defined on each page of site)
-- ex: <link rel="icon" href="favicon.ico">
- Recommended size - 48px square

Social Media Meta
- For SM crawlers like facebook to not have to guess the content of your website
- Facebook uses og properties
- Title of page:
-- <meta property="og:title" content="">
- Short description/summary of page:
-- <meta property="og:description" content="">
- Absolute url to an image that represents the content of your page. At least 600x315 pixels w/2:1 aspect ratio
-- <meta property="og:image" content="">
- Canonical URL for your page with any variables/parameters
-- <meta property="og:locale" content=""> 
- Ex:
    <meta property="og:title" content="Apple">
    <meta property="og:description" content="Discover the innovative world of Apple and shop everything iPhone, iPad, Apple Watch, Mac, and Apple TV, plus explore accessories, entertainment, and expert device support.">
    <meta property="og:url" content="https://www.apple.com/">
    <meta property="og:locale" content="en_US">
    <meta property="og:image" content="https://www.apple.com/ac/structured-data/images/open_graph_logo.png?202110180743">
    <meta property="og:type" content="website">
    <meta property="og:site_name" content="Apple">
- X (Twitter) uses X cards
-- twitter:card --> summary
-- twitter:title
-- twitter:description
-- twitter:image
- Ex:
    <meta name="twitter:url" content="https://www.microsoft.com/en-us">
    <meta name="twitter:title" content="Microsoft – AI, Cloud, Productivity, Computing, Gaming &amp; Apps">
    <meta name="twitter:description" content="Explore Microsoft products and services and support for your home or business. Shop Microsoft 365, Copilot, Teams, Xbox, Windows, Azure, Surface and more.">
    <meta name="twitter:card" content="summary">

async/await
- Code will do one thing while waiting for another thing to happen. Like multitasking
- Built on top of promises
- Ex:
    const myPromise = new Promise((resolve, reject) => {
        const success = true; // Simulate success or failure
        if (success) {
            resolve("Operation was successful!");
        } else {
            reject("Operation failed.");
        }
    });
- To handle result of promise, you can use .then() and .catch()
- Ex:
    myPromise
    .then((result) => {
        console.log(result); // Output: "Operation was successful!"
    })
    .catch((error) => {
        console.error(error); // Output: "Operation failed."
    });
- async/await can simplify these processes above ^^^
- Ex (of above example rewritten):
    const myAsyncFunction = async () => {
        try {
            const result = await myPromise; // Wait for the promise to resolve
            console.log(result); // Output: "Operation was successful!"
        } catch (error) {
            console.error(error); // Output: "Operation failed."
        }
    };
- Ex (using async/await to fetch data from API):
    const fetchData = async () => {
        try {
            const response = await fetch("https://jsonplaceholder.typicode.com/posts/1"); // Wait for the fetch to complete
            const data = await response.json(); // Wait for the response to be converted to JSON
            console.log(data); // Output the fetched data
        } catch (error) {
            console.error("Error fetching data:", error); // Handle any errors
        }
    };

Fetch API
- Personal delivery service to make network requests
- Ex:
    const response = await fetch('https://jsonplaceholder.typicode.com/todos/');
-- Needs to be inside an async function 
- After, you can use .json() method on response object to parse the JSON data and use into
- Ex:
    async function getData() {
        const response = await fetch('https://jsonplaceholder.typicode.com/todos/');
        const data = await response.json(); // parse the JSON data
        console.log(data);
    }

    getData();
